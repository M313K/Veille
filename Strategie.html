Les Six Stratégies de Migration vers le Cloud

Connues sous le nom des « Six R de la migration », voici les principales approches pour migrer vers le cloud :

1. Réhébergement (Rehosting)

Il s'agit de transférer L'infrastructure sur site vers le cloud sans modifications majeures. C'est une solution rapide pour un retour sur investissement immédiat. On pratique il s'agit de fournir le code source et c'est le fournisseur Cloud qui s'occupe de la cloudification

2. Nouvelle plate-forme (Replatforming)

Cette méthode permet d'optimiser l'environnement pour le cloud sans modifier l'architecture de base des applications.

3. Rachat (SaaS)

Elle consiste à remplacer une application par une solution SaaS équivalente, comme migrer un CRM vers Salesforce.

4. Redéveloppement (Refactoring)

Repenser complètement une application pour tirer parti des fonctionnalités avancées du cloud, comme l'évolutivité automatique.

5. Retrait (Retiring)

Supprimer les applications obsolètes qui ne sont plus nécessaires après une évaluation approfondie.

6. Conservation (Retaining)

Garder certaines applications sur site si la migration vers le cloud n’est pas encore adaptée à votre entreprise.
